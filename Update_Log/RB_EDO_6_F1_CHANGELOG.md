# RB_EDO_6_F1 更新记录

基线：`OpenRIAMap-Web_RB_EDO_6.zip`

## 修改范围

本次为 EDO_6 的小规模修复 patch，仅包含：

- 玩家信息卡右侧按钮位置微调；
- 移动端玩家卡导航按钮跳转修复；
- BUD/STB 面积内部显示优先级映射修复。

## 1. 玩家信息卡按钮位置微调

- 将桌面端玩家卡内“导航 / 分享”按钮组从 `top-[46px]` 上移到 `top-[36px]`。
- 将玩家 ID 文本区右侧预留从 `sm:pr-[96px]` 调整为 `sm:pr-[88px]`。
- 保持右侧 2×2 按钮区结构不变：第一行缩放/关闭，第二行导航/分享。
- 不修改其他 DraggablePanel 的默认按钮布局。

## 2. 移动端玩家导航修复

- 修复移动端玩家卡点击“导航”后仍停留在玩家卡内容的问题。
- 在移动端玩家导航按钮触发时清除 `selectedPlayer`，让移动端 sheet 正常切换到导航面板。
- 终点仍使用点击时玩家当前坐标，导航输入显示坐标值。
- 桌面端玩家卡导航逻辑保持不变。

## 3. BUD/STB 面积内部优先级修复

- 将 `STRUCTURE_LABEL_AREA_BONUS_DIVISOR` 从 `1000` 调整为 `10`。
- 保留 `STRUCTURE_LABEL_AREA_BONUS_MAX = 999`。
- BUD/STB 普通要素内部优先级现在按 `floor(area / 10)` 加分，并限制在 0–999。
- 特殊定义池优先级仍高于普通面积加权 BUD/STB。
- 外部 displayTier 和其他类别的整体显示优先级不变。
