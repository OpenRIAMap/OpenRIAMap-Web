# RB_EDO_4_F1 分享链接运行期与无效 ID 反馈修复

## 基线

- OpenRIAMap-Web_RB_EDO_4.zip

## 修改范围

- src/lib/featureShareLink.ts
- src/components/Map/MapContainer.tsx

## 修改内容

1. 将分享链接 dev StrictMode 内存兜底保留时间从即时清理调整为 1500ms，避免本地开发模式双挂载时丢失首次打开的分享目标或无效提示。
2. 在 MapContainer 中增加 hashchange / popstate 监听，使地图已打开时在地址栏输入新的分享链接也能重新消费。
3. 将“有效 world 但 ID 不存在”的等待逻辑从原先约 20 秒缩短为两阶段流程：
   - 等待搜索池准备：最多 12 次，每次 250ms，约 3 秒；
   - 搜索池有内容后查找目标：最多 8 次，每次 250ms，约 2 秒。
4. 增加局部分享链接查找加载条，用于提示“正在准备分享要素索引”或“正在查找分享要素”。
5. 找到目标后清理局部加载条并执行原有要素选择跳转；找不到目标后清理局部加载条并显示“无效世界或要素ID”。

## 不修改范围

- 不修改 Vercel / IIS 部署配置。
- 不修改 GitHub 数据源路径。
- 不修改玩家图层、玩家搜索、导航功能。
- 不修改 DraggablePanel、Toolbar、LoadingOverlay、loadingStore。
- 不修改 RuleDrivenLayer 渲染逻辑和 label / BUD / STB 显示逻辑。

## 核验建议

1. 首次打开 `/#/zth/有效ID`：URL 清理为 `/`，显示短暂查找条，并跳转到目标要素。
2. 首次打开 `/#/zth/不存在ID`：URL 清理为 `/`，显示查找条，并在约 3-5 秒内弹出“无效世界或要素ID”。
3. 当前已在 `/` 时，在地址栏输入 `/#/zth/有效ID`：不需要整页刷新，也能重新触发跳转。
4. 当前已在 `/` 时，在地址栏输入 `/#/zth/不存在ID`：不需要整页刷新，也能显示查找条并快速失败。
5. 刷新 `/`：不会重复触发上一条分享链接。
